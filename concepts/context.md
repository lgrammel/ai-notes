# Context

Context is the set of information an LLM application provides alongside a request (instructions, conversation history, retrieved documents, tool outputs, metadata) that the model can use to produce an answer.

## Examples

- System and developer instructions that define the assistant's role.
- A chat transcript and a short running summary of prior turns.
- Retrieved passages (RAG) and citations.
- Tool results (for example, database rows or API responses).
- PDF files and images.

## Synonyms

model context, input context (overlapping).
